# Task: Core compression engine

## Description

Implement the mechanical, deterministic `compress(state, options)` core: a single
recursive walker that caps depth, strips configured keys, caps array length,
optionally drops empty values, and is safe against circular references. No React, no
network, no models. This is the heart of the library and the structural contract that
sanitization (003) composes into.

## Acceptance Criteria

- [ ] `compress(state, options)` returns a new plain object; input is never mutated.
- [ ] `maxDepth` truncates deeper nodes to a documented placeholder (e.g.
      `"[Object]"`/`"[Array]"`), not recursing further.
- [ ] `strip` removes keys by exact string and by `RegExp` at any depth.
- [ ] `maxArrayLength` caps arrays and records the elided count (e.g. a
      `"[+N more]"` marker or metadata).
- [ ] `dropEmpty` flag removes `null`/`undefined`/`""`/`[]`/`{}` when enabled.
- [ ] Circular references are replaced with a marker instead of throwing.
- [ ] Same input + options always yields identical output (deterministic).
- [ ] `CompressOptions` is fully typed; defaults make `compress(state)` work with no
      config.

## Technical Details

- Implement in `src/core/compress.ts`; export from `src/index.ts`.
- Use a `WeakSet`/visited set for circular detection.
- Document handling of non-plain values (Date, Map, Set, functions) — predictable
  fallbacks (e.g. functions dropped, Date left as-is/ISO).
- O(n) over state nodes.
