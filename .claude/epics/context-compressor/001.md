# Task: Scaffold + toolchain

## Description

Stand up the package skeleton and the full build/test/lint/release toolchain (Stack 1:
tsup + Vitest + Biome + Changesets + size-limit) so every later task has working
`build`, `test`, `lint`, and `typecheck` commands. Create the two entry points (`.`
core and `./react` bindings) as empty-but-typed stubs that compile and emit ESM, CJS,
and `.d.ts` declarations.

## Acceptance Criteria

- [ ] `package.json` defines subpath `exports` for `.` and `./react`, `type: module`,
      ESM+CJS+types per entry, `sideEffects: false`, `react` as a peerDependency (>=17),
      and zero runtime `dependencies`.
- [ ] `tsup` config builds both entry points to `dist/` as ESM (`.mjs`/`.js`), CJS
      (`.cjs`), and `.d.ts`.
- [ ] `tsconfig.json` is `strict`, `isolatedModules`, `moduleResolution: bundler`,
      `declaration: true`.
- [ ] Vitest configured with `@vitest/coverage-v8`; a trivial passing test runs.
- [ ] Biome configured for lint + format; `lint` script passes clean.
- [ ] Changesets initialized; `size-limit` configured with a byte budget for each entry.
- [ ] GitHub Actions CI workflow: typecheck → lint → test+coverage → build → size-limit.
- [ ] `src/index.ts` (core stub) and `src/react/index.ts` (react stub) exist and build.

## Technical Details

- Build: tsup (esbuild). Test: Vitest. Lint/format: Biome. Versioning: Changesets.
  Bundle budget: size-limit. Document the test command in `CLAUDE.md` (no test
  framework existed before this task).
- Core stub exports placeholder `compress` signature + `CompressOptions` type; react
  stub re-exports a placeholder hook. Real logic lands in 002–004.
