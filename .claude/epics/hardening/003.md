# Task: Robustness hardening

## Description

Close the residual Low/Medium robustness findings from the Phase 1 security audit and
code reviews (SPLIT-PLAN §6 (backlog)). Touches the trust boundary, so this task gets
a security-auditor pass in addition to code review.

## Acceptance Criteria

- [ ] Map/Set iteration and array element reads are wrapped like the object path, so a
      Proxy-backed or throwing source degrades to a marker (e.g. `"[Getter]"`) instead
      of throwing — `compress` never crashes on a hostile node (tested).
- [ ] `Date` is deep-copied (`new Date(value.getTime())`) so the output never shares a
      live reference with the input (tested: mutating the output Date doesn't touch input).
- [ ] `redactedValue` is coerced to a string in `resolveOptions` (a non-string passed by
      a JS caller does not leak through as a raw value).
- [ ] A dev-only `console.warn` fires when `defaultSanitize:false` and `sanitize` is
      empty (redaction fully disabled); silent in production / when redaction is active.
- [ ] No public API change; coverage stays ≥ 90%.

## Technical Details

- Extend the existing try/catch + marker pattern in `src/core/compress.ts` to the
  Map/Set/array read paths. Date copy in the `instanceof Date` branch. Coercion in
  `resolveOptions`. Dev-warning gated on a `process.env.NODE_ENV !== "production"`
  check (guarded so it's tree-shakeable / no-op in prod).
