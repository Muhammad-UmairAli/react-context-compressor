# Task: Packaging guards + sourcemap trim

## Description

Guard the published package against exports-map / type-resolution regressions, and
trim sourcemaps from the tarball so the published bundle stays lean. CI-only tooling;
no runtime deps, no public API change.

## Acceptance Criteria

- [ ] `@arethetypeswrong/cli` (`attw --pack`) added as a CI step; passes on the
      current package and FAILS on a deliberately broken exports map (verified once).
- [ ] `publint` added as a CI step; passes clean.
- [ ] Published tarball no longer includes `*.map` (verify via `npm pack --dry-run`);
      sourcemaps still emitted for local dev builds OR excluded from `files`.
- [ ] `size-limit` still green; bundle unchanged or smaller.
- [ ] `attw` / `publint` available as `npm run` scripts for local use.

## Technical Details

- Add `@arethetypeswrong/cli` + `publint` dev deps and `attw` / `publint` scripts.
- Drop maps from the package: either `sourcemap: false` for the published build or
  exclude `*.map` from `files`/add an `.npmignore`. Keep `attw`/`publint` in the
  `build` CI job after `npm run build`.
